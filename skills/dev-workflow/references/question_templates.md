# question 工具模板

> 本文件收录 dev-workflow 各场景的 `question` 工具调用模板。所有调用默认启用 `custom: true`（用户可输入自由文本），以下示例省略该字段。

## 阶段文档确认

用于各阶段文档的审阅确认。使用时将 `【阶段名称】` 替换为实际阶段名，`header` 替换为对应的审阅名称，确认选项的 description 填入下一阶段名称。

```json
{
  "questions": [
    {
      "question": "请审阅【阶段名称】文档。是否确认？",
      "header": "文档审阅",
      "options": [
        {"label": "确认，进入下一阶段", "description": "文档内容无误，推进到下一个阶段"},
        {"label": "需要修改", "description": "文档需要调整，指出修改内容后重新确认"}
      ]
    }
  ]
}
```

## 设计稿方式选择

用于阶段 2 选择设计稿获取方式：

```json
{
  "questions": [
    {
      "question": "请选择设计稿的获取方式。",
      "header": "设计稿",
      "options": [
        {"label": "提供设计稿链接或图片", "description": "请在自定义输入中粘贴 Figma 等设计稿链接，或提供 PNG 图片路径，AI 将记录并保存设计资源"},
        {"label": "由 AI 生成设计稿", "description": "AI 根据需求文档和设计思路，自动生成设计文档和设计稿图片（PNG）"},
        {"label": "跳过设计稿", "description": "项目暂无设计稿，直接进入需求拆解阶段"}
      ]
    }
  ]
}
```

## 技术方案选择确认

用于阶段 4 在多个技术方案中选定其一。当方案超过 2 个时，按相同模式追加"采用方案C"等选项。

```json
{
  "questions": [
    {
      "question": "请审阅技术方案，并在以下方案中选择其一。",
      "header": "技术方案选择",
      "options": [
        {"label": "采用方案A", "description": "采纳方案A，进入实现清单阶段"},
        {"label": "采用方案B", "description": "采纳方案B，进入实现清单阶段"},
        {"label": "需要修改", "description": "方案需要调整，请在自定义输入中说明修改内容"}
      ]
    }
  ]
}
```

## 迭代命名确认

用于迭代步骤 1 确认迭代名称：

```json
{
  "questions": [
    {
      "question": "请确认本次迭代名称：【提议名称】",
      "header": "迭代命名",
      "options": [
        {"label": "确认名称", "description": "采用提议的迭代名称，继续确认受影响阶段"},
        {"label": "需要修改", "description": "在自定义输入中提供新的迭代名称"}
      ]
    }
  ]
}
```

## 迭代影响范围确认

用于迭代步骤 1 确认受影响的阶段列表：

```json
{
  "questions": [
    {
      "question": "本次迭代将更新以下阶段：【阶段列表】。是否正确？",
      "header": "影响范围",
      "options": [
        {"label": "确认，开始迭代", "description": "受影响阶段列表正确，开始逐阶段确认"},
        {"label": "需要调整", "description": "增删受影响阶段，在自定义输入中说明调整内容"}
      ]
    }
  ]
}
```

## 工作流完成

用于工作流完成后的后续操作：

```json
{
  "questions": [
    {
      "question": "工作流已完成！接下来想做什么？",
      "header": "后续操作",
      "options": [
        {"label": "提出新需求变更", "description": "进入迭代流程，对现有项目进行修改"},
        {"label": "启动新项目", "description": "重置工作流，从零开始一个新项目（将清除当前项目的全部文档与代码，需二次确认）"}
      ]
    }
  ]
}
```

> **重要：** 如果用户选择"启动新项目"，**不得立即执行**。必须先进行二次风险告知与确认（参照下文「新项目风险确认」），只有用户在二次确认中明确选择"确认删除并重启"后，才执行「重新开始」流程。

## 新项目风险确认

用于用户选择"启动新项目"后的二次风险告知与确认。**必须先告知后果，再由用户二次确认，确认前不得删除任何文件。**

```json
{
  "questions": [
    {
      "question": "⚠️ 即将清除当前项目的全部内容并重启工作流。这将永久删除以下目录及其所有文件：\n.dwf/、01-需求文档/、02-设计稿/、03-需求拆解文档/、04-技术方案/、05-实现清单/、06-code/、07-需求迭代/\n\n此操作不可撤销，现有的需求文档、设计稿、技术方案、实现清单和全部代码都将丢失。是否确认？",
      "header": "新项目风险确认",
      "options": [
        {"label": "确认删除并重启", "description": "已了解后果，永久删除当前项目所有内容并从零开始新项目"},
        {"label": "取消，保留现有项目", "description": "不删除任何内容，回到当前工作流"}
      ]
    }
  ]
}
```
