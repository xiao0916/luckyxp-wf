# 使用指南设计文档

- **日期**: 2026-06-25
- **主题**: 为 skills 目录下两个技能（animation-flow、dev-workflow）编写终端用户使用指南
- **状态**: 已确认

## 目标

为两个技能各写一份面向终端用户（与 AI 对话的人）的快速上手指南，帮助用户快速知道：怎么触发、要给什么、每步会发生什么、要确认什么。

## 决策

| 决策项 | 选择 |
|--------|------|
| 目标读者 | 终端用户（和 AI 对话的人），偏「怎么用」 |
| 输出形式 | Markdown 文件，随技能存放：`skills/<skill>/USAGE.md` |
| 详细程度 | 快速上手型（精简） |
| 语言 | 中文 |
| 结构方案 | 方案 A：触发词优先 + 效果/阶段速查表 + 一个端到端示例 + 注意事项 |

## 结构方案对比

- **方案 A（采用）**：触发词表 → 速查表（每行：触发示例 / 要给什么 / AI 做什么 / 要确认什么）→ 一个完整示例 → 注意事项。可扫读、即查即用，最贴合精简目标。
- 方案 B（场景驱动）：篇幅偏长，与精简目标冲突。
- 方案 C（FAQ）：缺乏全局结构，首次阅读不友好。

## 交付物

1. `skills/animation-flow/USAGE.md`
   - 这是什么 → 怎么触发（5 类效果关键词）→ 五类效果速查表 → 一次完整用法（数字滚动示例）→ 触发不到怎么办 → 你需要注意 → 想加新效果
2. `skills/dev-workflow/USAGE.md`
   - 这是什么 → 怎么触发 → 六个阶段速查表 → 关键机制（确认/方案选择/设计稿选项/状态恢复）→ 一次完整用法 → 已有代码后想改东西（迭代）→ 想从头开始 → 你需要注意

## 不做

- 不写面向技能开发者的内部机制/扩展文档（用户已明确只需终端用户视角）。
- 不做完整走查式参考手册（已选精简型）。
